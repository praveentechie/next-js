## 🧠 Learning & Knowledge Base

This repository serves as a personal "living" textbook. I use a standardized tagging pattern in code comments to make lessons learned easily searchable across all tech stacks.

### 🏷️ The Pattern
`[TECH] [CATEGORY]: [MESSAGE]`

| Tag | Purpose | Search Example |
| :--- | :--- | :--- |
| **`LEARN`** | New syntax or concepts discovered. | `[JS] LEARN` |
| **`WHY`** | Rationale for a specific implementation. | `[PY] WHY` |
| **`GOTCHA`** | Unexpected behaviors or fixed bugs. | `[REACT] GOTCHA` |
| **`REF`** | Links to documentation or articles. | `[SQL] REF` |
| **`BACKLOG`** | Concepts to research deeper later. | `[GO] BACKLOG` |

### 🔍 How to Search
Use your IDE's global search (**Ctrl+Shift+F**) or `grep` from the terminal:
*   **Filter by Stack:** Search for the tech prefix (e.g., `[JS]` or `[PY]`).
*   **Filter by Lesson Type:** Search for the category (e.g., `LEARN:` or `GOTCHA:`).
*   **Specific Search:** Combine them for precise results (e.g., `[REACT] LEARN`).
